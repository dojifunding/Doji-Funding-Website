# .impeccable — Doji Funding Design Standard

## Who we're designing for

All traders, regardless of level. A first-timer discovering prop trading sits next to a veteran allocating serious capital. The interface must never condescend to beginners nor bore professionals. Information density should be adjustable by context, not assumed by the designer.

## Brand personality

**Customisable. Transparent. Professional.**

- **Customisable**: The platform bends to the trader, not the other way around. The configurator is a first-class feature, not a checkout flow. Options are exposed, not hidden.
- **Transparent**: Every number shown is justified. No dark patterns. Pricing is clear. Rules are explicit. Nothing is hidden behind a CTA.
- **Professional**: The aesthetic is that of a serious financial tool — not a game, not a landing page, not a lifestyle brand. Traders trust terminals. We are closer to a terminal than a startup.

## Emotional target

When a user opens the dashboard they must feel:

1. **Confidence** — this firm is real, established, and serious. Clean layout, precise typography, no visual noise.
2. **Precision** — every number, every status, every session indicator is accurate and immediately readable. Nothing is decorative if it carries data.
3. **Comfort** — whatever their level, the interface supports them. No intimidation, no jargon without context, no feature that requires prior knowledge to find.

## What Doji Funding is NOT

Every major competitor (FTMO, Funding Pips, FundedNext, Bright Funded, e8 Markets, The5ers) converged on the same aesthetic: hero gradient, icon grid of 3–4 stats, identical card layout, same green accent. They are visually interchangeable.

Doji Funding must be immediately distinguishable at a glance — not by being louder, but by being more deliberate.

**Avoid:**
- Gradient text on headings (used by every competitor)
- Icon boxes in stat cards (decorative, not informative)
- Symmetrical 3 or 4-column "benefit" grids with icons (universal AI slop)
- Generic "Start your journey" / "Get funded today" hero copy
- Any element that could exist on a competitor's site unchanged

## Design rules derived from these values

### Typography hierarchy (non-negotiable)
- **Doto** — large display numbers only (prices, stats, % splits). Never for labels or body copy.
- **Chivo Mono** — all UI chrome: labels, nav items, buttons, table headers, form fields, status tags.
- **Nippo** — body text, descriptions, explanatory copy. Never for data.
- No other fonts. No mixing within a single element.

### Colour discipline
- `#000000` — page background (OLED black). No substitutes.
- `#050505` / `#111111` — card and surface backgrounds only.
- `#10B981` — one accent. Used for: active states, positive values, CTAs, progress indicators.
- `#D71921` — reserved exclusively for errors and destructive actions. Never for decoration.
- White (`#FFFFFF`) at varying opacities for text hierarchy.
- **No other accent colours.** No gradients. No glow animations on data elements.

### Spacing and geometry
- 2px border-radius maximum. Square is intentional — precision tool, not consumer app.
- Borders: 1px solid `rgba(255,255,255,0.08)` for surfaces. Never decorative dividers.
- Consistent 8px base unit. Spacing: 8, 12, 16, 24, 32, 48.

### Data display principles
- Numbers are the hero. Typography does the work. Icons are not needed to identify a stat.
- If a number needs an icon to be understood, the label is insufficient — fix the label.
- Session state (open/closed), challenge status, payout amounts: always immediately scannable without hover.

### Component tone
- Buttons: uppercase, Chivo Mono, no rounded corners, no shadows.
- Status tags: Chivo Mono, minimal padding, 2px border or solid fill.
- Cards: flat background, 1px border, no drop shadows. Elevation through border contrast, not shadow.
- Tables: no zebra striping with decorative colour. Row separation via border only.

### Anti-patterns to catch in review
- Any `box-shadow` in `@keyframes` (repaint every frame)
- `filter: blur()` on reveal animations (costly, rarely needed)
- `cubic-bezier` with overshoot (bounce) — use deceleration curves only (`0.22, 1, 0.36, 1`)
- Hardcoded colour values in SVG attributes (`stroke="#10B981"`) — always use `currentColor`
- Google Fonts not in the active font list loaded in `<head>`
- Orphaned CSS rules (selector missing, closing brace without opening)

## Differentiator to protect

The Nothing OS-inspired aesthetic is Doji Funding's clearest visual differentiator from every competitor. It was a deliberate choice. Any new component, page, or feature must be compatible with this system — not grafted on top of it.

When in doubt: reduce, don't add. Precision over decoration. Data over imagery. Monospace over sans-serif.
